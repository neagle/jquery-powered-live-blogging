Starting Over

jQuery Plugin

  • Very simple pattern
  • AOL developers are used to working with them
  • We were used to writing them
$.fn.myWidget = function () {
  return this.each(function () {
    $(this).append('<p>Hello world!</p>');
  });
};

$('p').myWidget();

Two plugins, actually.

  • $.liveUpdateApi
  • $.liveUpdateUi

$.liveUpdateApi

$.fn.liveUpdateApi = function () {
  return this.each(function () {
    // Hit the Live Update API with an AJAX call

    $(this).trigger('update', {
      // Data from the API
    });
  });
});

$.liveUpdateUi

$.fn.liveUpdateUi = function () {
  return this.each(function () {
    var $this = $(this);

    $this.liveUpdateApi();
    $this.bind('update', function (data) {
      // Build some UI with the incoming data
    });
  });
});

The API Plugin

Method / Instantiation Pattern

var methods = {
  init: function (options) {},
  fetch: function () {},
  pause: function () {},
  resume: function () {}
};

return this.each(function () {
  if (methods[method]) {
    return methods[method].apply(this, Array.prototype.slice.call(arguments, 1));
  } else if (typeof method === 'object' || ! method) {
    return methods.init.apply(this, args);
  } else {
    $.error('Method ' + method + ' does not exist on jQuery.liveBlogApi.');
  }
});

URL Pattern

  • postId/0
    The initial call grabs everything.
  • In that call’s data is a last_update value.
  • postId/12432
    For subsequent calls, use the last_update value in the URL.
  • Only new information is sent.
  • Keep using the same URL until the value of last_update changes.

Caching

  • We want to decrease the impact of so many requests.
  • If we cache the URLs, we need a way to break cache when there’s new information.
  • So we increment the callback value of the AJAX request.
http://liveupdater.aol.com/live-update/20334012/33883?callback=LB_U33883_C0
http://liveupdater.aol.com/live-update/20334012/33883?callback=LB_U33883_C1
http://liveupdater.aol.com/live-update/20334012/33883?callback=LB_U33883_C2
http://liveupdater.aol.com/live-update/20334012/33883?callback=LB_U33883_C3
http://liveupdater.aol.com/live-update/20334012/33884?callback=LB_U33883_C0

Warning! For jsonp requests, the cache setting on $.ajax defaults to false.

  • Update them
  • Delete them
  • Make it all look pretty

Actually, we need a lot more.

Lessons Learned

Starting over might be easier than it looks.

But if you can, don’t make the next person do it.

Optimization

  • Start / stop polling when the live blog is on / off screen
  • Only load images when visible
  • Just-in-time loading for Tweet buttons

Sometimes, perceived load time is more important than load time.

  • setTimeout(0)
  • Load updates in chunks

Iteration and Observation

  • Updates notification
  • Image resizing
  • Featured authors
  • Embedded HTML

Simplicity is good, but it can have its limitations.

We could use some views.
