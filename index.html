jQuery Code Styles

  • Declarative
  • Dynamic
  • Contextual

Declarative

  • Heavy use of IDs
  • All traditional event binding
  • Often repeated code with slight variations
  • Changes to HTML often require changes to JS and vice versa
  • Large number of selectors running on Document Ready
$(function () {
    var $container = $( '#container' ),
        $slider = $( '#slider' ),
        $footer = $( '#footer' );

    $( "#toggle" ).click(function () {
        $container.toggle();
    });

    $slider.click(function () {
        $footer.html( $slider.html() );
    });
});

Dynamic

  • Heavy use of classes, simple selectors
  • Mixture of traditional binding and live/delegate binding
  • Less repeated code
  • Changes to HTML may still require change to JS
  • Many selectors still running on Document Ready
$(function () {
    var $container = $( '#container' );

    $( 'a.toggle' ).live( 'click', function( e ) {
        $container.toggle();
        e.preventDefault();
    });
});

Contextual

  • Heavy use of selectors and traversing
  • Minimal use of IDs or classes
  • Very little repeated code
  • HTML follows a convention so edits require fewer changes to the JS
  • Virtually no selectors running on Document Ready
$( 'form a.submit' ).live( 'click', function( e ) {
    e.preventDefault();
    $( this )
        .closest( 'form' )
        .find( '.errors' ).slideUp().end()
        .submit();
});

Benefits of Contextual jQuery

Benefits

  • Faster, flexible, and more responsive code
  • Focused use of resources
  • Less code
  • Easier to maintain
  • Easier to reuse

Live Events

Bound Events

$( 'a' ).click( ... );

Live Events

$( 'a' ).live( 'click',  ... ); // 1.3+
$( document ).on( 'click', 'a', ... ); // 1.7+

Delegate Events

$( '#container' ).delegate( 'a', 'click',  ... ); // 1.4.3+
$( '#container' ).on( 'click', 'a', ... ); // 1.7+
Bind Delegate
Separate handler for each element Single handler for all elements
Executes only when event is triggered on bound elements Checks event on every element in its context, and executes when a match is found
Elements can be retrieved ahead of time Elements should be retrieved at run time
Must be bound after the DOM is ready Can be set up at any time

What about initialization?

  • Keep the setup minimal
    • Use CSS and js / no-js classes
    • Use selectors or data to determine initialization state
  • Always opt for just-in-time initialization

js / no-js

<!doctype html>

<head>

    <style>
        html.no-js .js, html.js .no-js { display: none; }
    </style>
    <script>
        document.documentElement.className =

Determine Initialization State

$( 'a[href^=contact]' ).live( 'click', function ( event ) {
    var dialog = $( '#dialog' );
    if ( dialog.data( 'dialog' ) ) {
        dialog.dialog( 'open' );
    } else {
        dialog.dialog();
    }
    event.preventDefault();
});

$( 'a[href^=contact]' ).live( 'click', function ( event ) {
    var dialog = $( document ).data( 'dialog' ) ||
                 $( document )
                    .data( 'dialog', $( '#dialog' ).dialog( { autoOpen: false } ) )
                    .data( 'dialog' );

    dialog.dialog( 'open ' );
});

Opt to pay a little more later so you don’t risk losing it all on something that never happens.

Not vs. Is

if ( $( 'a' ).not( 'a' ) ) {
    alert( 'Why does this work?' );
}
            
if ( $( 'a' ).is( ':not( a )' ) {
    alert( 'This won\'t show... phew!' );
}

if ( ! $( 'a' ).is( 'a' ) ) {
    alert( 'This won\'t show either!' );
}

HTML Conventions

Semantic HTML

<div>
    <p class="post-title">My Article</p>
    <p class="post-author"><del>Doug Neiner</del> Nate Eagle</p>

    <p>Hi there!</p>
    <p class="post-quote">My quote!</p>
</div>

<div class="post">
    <h2>My Article</h2>
    <cite><del>Doug Neiner</del> Nate Eagle</cite>

    <p>Hi there!</p>
    <blockquote>My quote!</blockquote>
</div>

Conventions are Patterns

  • You can build them yourself
  • They need to be consistent (or they are’t patterns)
  • They are a promise you make
  • They can change between projects
  • They need to work for you!

Don’t think of IDs and classes as your first line of attack!

Write code like you ask for directions.

Try to get there on your own first!

In review

  • Don’t spend resources early, strive for just-in-time wherever possible.
  • Don’t be afraid to use complex selectors in the right settings.
  • Think twice before adding IDs or classes to your markup. See if there is another way first.
  • Always write your code with reusability in mind. Think in conventions as you work.
